Fault Injection in Surface Codes

Validating Quantum Reliability through Logical Qubit Scaling and Error Suppression

View Implementation Methodology

System Architecture

This project focuses on the implementation and validation of the Rotated Surface Code. The primary objective is to demonstrate how increasing the "Code Distance" (redundancy) impacts the logical error rate of the system.

The Lattice Design

The architecture utilizes a 2D grid of physical qubits arranged to perform simultaneous parity checks:

  • Data Qubits: Holders of the quantum information.
  • Syndrome Qubits: Ancilla qubits used to measure X and Z stabilizers without collapsing the superposition.

Validation Strategy: Fault Injection

To strictly satisfy the course requirement for Reliability Validation, this project employs a fault injection strategy. Instead of waiting for natural environmental errors, we deliberately inject noise into the simulation to stress-test the decoder.

Phase 1: Detector Error Models

Generating a graph of potential fault mechanisms (Pauli X/Z errors) to verify that the decoding logic can identify error chains.

Phase 2: Artificial Noise Injection

Applying unitary rotations (Coherent Errors) and reducing T1/T2 times during idle cycles in the simulation. This validates the "Threshold Theorem" by observing system behavior under high-stress conditions.

Phase 3: Decoder Verification

Running the Minimum Weight Perfect Matching (MWPM) algorithm against the injected faults to calculate the logical fidelity.

Implementation Framework

The validation environment is being constructed using CUDA-Q to accelerate the circuit simulations. Below is a representation of the kernel structure used for the fault injection loop.

// CUDA-Q Kernel for Fault Injection
auto surface_code_kernel = [](int distance, double noise_prob) {
  cudaq::qvector qubits(2 * distance * distance - 1);

  // 1. Initialize Logical Zero
  h(qubits);

  // 2. Inject Faults (Identity + Noise Channel)
  for (int i = 0; i < qubits.size(); i++) {
    if (random_double() < noise_prob) {
      x(qubits[i]); // Inject Bit Flip
    }
  }

  // 3. Syndrome Extraction Cycle
  measure_stabilizers(qubits);
};
Current Status: The simulation kernel is being optimized to run varying Code Distances (d=3, d=5) to gather data on the logical error suppression rates.
